# AGENTS.md

이 저장소에서 Codex는 아래 원칙을 따른다.

## 1) 우선순위
- 사용자의 최신 요청을 가장 먼저 따른다.
- 변경은 작고 명확하게 유지한다.

## 2) 코드 작성 원칙
- 가독성과 단순성을 최우선으로 한다.
- 사람이 빠르게 이해할 수 있는 구조와 이름을 사용한다.
- 복잡하거나 난해한 최적화는 피한다.
- 성능보다 유지보수성이 중요한 경우 기능을 단순화한다.
- 기존 코드 스타일과 패턴을 따른다.

## 3) 변경 범위
- 요청과 직접 관련된 파일만 수정한다.
- 불필요한 리팩터링, 이름 변경, 포맷 대규모 수정은 하지 않는다.
- 새 의존성은 꼭 필요할 때만 추가한다.

## 4) 검증
- 변경 후 가능한 가장 작은 범위의 검증을 수행한다.
- 테스트/린트 실행이 불가하면 이유를 명확히 보고한다.

## 5) 커뮤니케이션
- 결과는 간결하게 보고한다: 무엇을 바꿨는지, 어떤 검증을 했는지, 남은 리스크는 무엇인지.

## 6) 본 프로젝트 맥락 (PRD 기준)
- 목표: 한국어 음성 명령 텍스트의 intent 분류와 slot 추출.
- 기본 원칙: 정확성과 예측 가능성 우선, 과도한 추론 로직보다 명시적 규칙/스키마를 선호.
