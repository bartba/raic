RAIC Code Overview

현재까지 생성된 코드를 초보 엔지니어 관점에서 읽기 쉽게 정리한 구조도입니다. 핵심은 API 입구, 데이터/모델, 스키마 로더, 테스트의 관계입니다.

Entry

앱 시작점

api/main.py가 FastAPI 앱을 만들고 /health 라우터를 붙입니다.

Models

공용 타입

api/models/common.py에 Decision, Confidence, SlotType를 모았습니다.

Schema

YAML 로더

api/services/schema_manager.py가 data/intents.yaml과 data/devices.yaml을 읽고 검증합니다.

Tests

안전장치

tests/unit/이 설정, 모델, YAML, 스키마 로더를 각각 좁게 검증합니다.

Structure Flow

앱 진입점 모델 파일 스키마 로더 테스트

What Changed So Far

얇은 앱 골격

main.py와 health.py만으로 서버 기동 경로를 먼저 만들었습니다.

데이터 우선

분류 로직보다 먼저 intents.yaml, devices.yaml, golden을 정의했습니다.

검증 중심

YAML 파싱, Pydantic 검증, 중복 검사, 스키마 조회를 unit test로 고정했습니다.

단순화 포인트

공용 타입은 common.py로 모으고, 스키마 로드는 load_schema() 한 진입점으로 정리했습니다.

테스트 축

test_config, test_models, test_data_files, test_schema_manager가 각 층을 좁게 검증합니다.
